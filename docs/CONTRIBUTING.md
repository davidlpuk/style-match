# Contributing to Style Match

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/style-match.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature-name`
5. Make your changes
6. Test your changes: `npm run lint && npm test`
7. Commit: `git commit -m "feat: your feature description"`
8. Push: `git push origin feature/your-feature-name`
9. Create a Pull Request

## Commit Convention

We follow the Conventional Commits specification:

- `feat:` New features
- `fix:` Bug fixes
- `docs:` Documentation changes
- `style:` Code style changes (formatting, etc.)
- `refactor:` Code refactoring
- `test:` Adding or updating tests
- `chore:` Maintenance tasks

## Code Style

- Use TypeScript for all new code
- Follow the ESLint configuration
- Write meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

## Testing

- Write unit tests for utilities and functions
- Add integration tests for API routes
- Test UI components with React Testing Library

## Pull Request Process

1. Update documentation if needed
2. Ensure all tests pass
3. Request review from maintainers
4. Address any feedback
5. Wait for approval and merge

Thank you for contributing!
